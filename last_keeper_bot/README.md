# Telegram-бот «Последний хранитель»

Рабочая тестовая версия цифрового Архивариуса для проекта 16–17 ноября 2026 года в Доме Москвы в Ереване.

## Реализовано

- регистрация и согласие на обработку данных;
- выбор дня участия;
- равномерное распределение по пяти командам;
- персональные маршруты команд;
- роли участника, капитана и администратора;
- открытие четырёх локаций по кодам;
- два подтверждаемых командных решения на каждой локации;
- скрытые параметры Архива и пять финальных архетипов;
- «Мешочек памяти»;
- обращения к организаторам;
- назначение капитана;
- одновременное открытие финала;
- CSV-экспорт участников;
- сохранение данных в SQLite после перезапуска;
- Docker-запуск.

## Безопасность

Токен Telegram не должен попадать в GitHub. Токен, отправленный в переписке, необходимо отозвать через BotFather и выпустить новый. Новый токен записывается только в локальный `.env` или в секреты хостинга.

## Запуск

```bash
cd last_keeper_bot
cp .env.example .env
```

Заполните в `.env`:

```env
BOT_TOKEN=новый_токен_из_BotFather
ADMIN_IDS=ваш_telegram_id
```

Затем:

```bash
docker compose up --build -d
docker compose logs -f
```

Для запуска без Docker:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python bot.py
```

## Первый тест

1. Отправьте боту `/start` и пройдите регистрацию.
2. Узнайте Telegram ID тестового участника.
3. Администратор назначает капитана командой `/captain TELEGRAM_ID`.
4. Капитан нажимает «Следующая точка» и вводит код локации.
5. Тестовые коды задаются в `.env`.
6. После четырёх решений администратор открывает общий финал через `/admin`.

## Команды администратора

- `/admin` — сводка, финал и экспорт;
- `/captain TELEGRAM_ID` — назначить капитана.

## Ограничения текущей версии

Это рабочий MVP для немедленного сценарного тестирования. Для промышленной эксплуатации на 150 участников необходимо следующим этапом перенести базу на PostgreSQL, FSM — на Redis, добавить временные окна кодов, полноценное редактирование маршрутов в админ-панели, журнал изменений, ответы администратора через бот, входное/итоговое измерение и автоматические тесты.

# Telegram-бот «ЭРА»

Облачный Telegram-бот общественной организации «ЭРА» — Объединение лидеров и культурных инициатив.

Текущая версия работает только внутри Telegram. Mini App отключён: все сценарии выполняются сообщениями и обычными Telegram-кнопками.

## Возможности

- регистрация участника и проверка подписки на канал;
- рассмотрение заявок администратором;
- личный кабинет, профиль, баллы, рейтинг и портфолио;
- мероприятия, регистрация, напоминания и подтверждение посещения;
- проекты с ручным или ИИ-оформлением;
- задачи участников;
- департаменты и направления;
- вопросы команде ЭРА;
- панели лидера и администратора;
- рассылки, роли, статусы, знаки отличия и аудит действий;
- работа в общем, департаментских и лидерском чатах;
- мягкая модерация и служебные команды.

## Основные команды

- `/start` — начало работы или главное меню;
- `/menu` — главное меню;
- `/rules`, `/links`, `/departments`, `/events`, `/project` — команды сообщества;
- `/leader` — панель лидера;
- `/admin` — панель администратора.

## Облачный запуск

Проект разворачивается на Render из `render.yaml`. Инфраструктура включает:

- Docker web service для Telegram webhook;
- PostgreSQL;
- Redis-совместимое хранилище Render Key Value;
- автоматический деплой после изменений в GitHub.

Секреты `BOT_TOKEN` и `WEBHOOK_SECRET` хранятся только в Render и не записываются в GitHub.

## Настройка Telegram

1. Добавьте бота администратором официального канала ЭРА для проверки подписки.
2. Добавьте бота в общий, департаментские и лидерский чаты.
3. Для модерации чатов отключите Privacy Mode через BotFather.
4. Заполните числовые ID канала и чатов в переменных Render.
5. Укажите Telegram ID администраторов в `ADMIN_IDS` через запятую.

## Локальный запуск

```bash
cp .env.example .env
docker compose up --build -d
```

Контейнер применяет миграции, запускает webhook, планировщик напоминаний и обработчики бота.

## Проверка

```bash
python -m unittest discover -s tests -v
```

GitHub Actions автоматически повторяет тесты после каждого обновления основной ветки.

## Безопасность

- не сохраняйте токен бота в репозитории;
- ограничивайте права администратора доверенными аккаунтами;
- согласуйте политику обработки анкет, телефонов и селфи;
- настройте резервные копии PostgreSQL перед постоянной эксплуатацией;
- проверяйте новые сценарии на небольшой тестовой группе.

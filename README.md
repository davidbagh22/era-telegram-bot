# Telegram-система «ЭРА»

Запускаемый MVP Telegram-бота общественной организации «ЭРА» — Объединение лидеров и культурных инициатив.

Система фиксирует путь участника от первой регистрации до проектов и лидерской ответственности: подписку, анкету, департаменты, мероприятия, посещения, баллы, рейтинг, задачи, проекты, портфолио и достижения.

## Что реализовано

- Telegram Mini App с адаптивным интерфейсом в фирменных цветах ЭРА;
- вход через Telegram с серверной проверкой подписи `initData`;
- единый FastAPI-сервис для Mini App API и Telegram webhook;

- обязательная проверка подписки на канал через `getChatMember`;
- многошаговая регистрация и согласие на обработку данных;
- рассмотрение заявок администратором;
- раздельные роль и статус участия;
- два департамента и шесть финальных направлений;
- личный кабинет, профиль и «Мой путь в ЭРА»;
- история баллов, рейтинг и знаки отличия;
- портфолио и текстовое резюме ЭРА;
- мероприятия, лимиты мест и регистрация;
- напоминания за 24 часа, 3 часа и 30 минут;
- подтверждение планов «приду / не смогу»;
- селфи в день мероприятия, ручная проверка, начисление баллов;
- обратная связь после подтверждённого посещения;
- заявки в департаменты;
- 12-шаговый конструктор проектов;
- ИИ-оформление проектов и концепций мероприятий;
- вопросы команде ЭРА и ответы администратора;
- панель лидера с ограничением по его департаментам и направлениям;
- создание мероприятий, задач, отчётов и предложений лидером;
- панель администратора: заявки, проекты, мероприятия, селфи, задачи, баллы, знаки, роли, статусы, вопросы и аналитика;
- сегментированные рассылки всем, по роли, департаменту или направлению;
- рассылки лидеров только после решения администратора;
- работа в общем, внутреннем, внешнем и лидерском чатах;
- команды `/rules`, `/links`, `/departments`, `/events`, `/project`;
- мягкая модерация флуда и личных выпадов;
- журнал значимых действий `audit_logs`;
- PostgreSQL, Redis FSM, Alembic, APScheduler и Docker Compose.

В соответствии с ТЗ здесь нет новостей, «Импульса ЭРА», экспертных разделов, отдельного раздела ЭРА PRO, PDF-генератора, магазина, QR-кодов, автоматического повышения ролей и ИИ-решений за администратора.

## Быстрый запуск

Нужны Docker Desktop и созданный через BotFather Telegram-бот.

1. Скопируйте `.env.example` в `.env`.
2. Заполните токен, ID канала, ID чатов и `ADMIN_IDS`.
3. Запустите:

```bash
docker compose up --build -d
```

4. Посмотрите состояние:

```bash
docker compose ps
docker compose logs -f bot
```

При старте контейнер собирает Mini App, применяет миграции, создаёт справочники и запускает FastAPI. Telegram присылает обновления через защищённый webhook.

## Облачное развёртывание на Render

Файл `render.yaml` описывает всю инфраструктуру:

- постоянный Docker web service с Mini App, API и Telegram webhook;
- управляемый PostgreSQL;
- совместимое с Redis хранилище Render Key Value;
- автоматический деплой после изменений в GitHub;
- секреты, которые вводятся только в панели Render и не сохраняются в репозитории.

В Render выберите **New → Blueprint**, подключите GitHub-репозиторий и укажите этот проект. При первом создании Render запросит значения переменных с `sync: false`, включая `BOT_TOKEN`, `OPENAI_API_KEY` и числовые ID Telegram-канала и чатов.

Render выдаёт приложению HTTPS-адрес. Он используется одновременно для Mini App и Telegram webhook. После каждого коммита в подключённую ветку Render автоматически пересобирает и перезапускает сервис.

## Обязательная настройка Telegram

### Официальный канал

Добавьте бота администратором официального канала ЭРА. Иначе Telegram не позволит надёжно проверять подписку через `getChatMember`.

Укажите числовой ID или публичное имя канала в `ERA_CHANNEL_ID`. Ссылка-приглашение сама по себе не является ID чата.

### Чаты

Добавьте бота администратором в:

- общий чат ЭРА;
- чат внутренних связей;
- чат внешних связей;
- чат лидеров.

Для приветствий и модерации отключите Privacy Mode у BotFather. Выдайте права отправлять, закреплять и при необходимости удалять сообщения.

Запишите числовые ID чатов в соответствующие переменные `.env`. Обычно ID супергруппы начинается с `-100`.

### Первый администратор

В `ADMIN_IDS` перечислите личные Telegram ID администраторов через запятую:

```env
ADMIN_IDS=123456789,987654321
```

Администратор проходит обычную регистрацию. Для ID из `ADMIN_IDS` заявка подтверждается сразу и назначается роль `admin`.

## OpenAI

ИИ-функции используют асинхронный клиент и Responses API. Укажите:

```env
OPENAI_API_KEY=...
OPENAI_MODEL=gpt-5.5
```

Модель можно заменить любой доступной текстовой моделью. Ключ хранится только в `.env`.

Если ключ не задан или API временно недоступен, проектный конструктор не падает: он формирует структурированный черновик без ИИ. Остальные функции бота полностью независимы от OpenAI.

## Команды

Пользовательские:

- `/start` — вход и проверка подписки;
- `/menu` — главное меню;
- `/rules`, `/links`, `/departments`, `/events`, `/project` — команды в чатах.

Лидерские:

- `/leader` — панель лидера.

Административные:

- `/admin` — панель администратора;
- `/addpoints Telegram_ID количество причина`;
- `/awardbadge Telegram_ID | название знака | причина`;
- `/portfolio Telegram_ID | тип | название | описание` — команду можно отправить ответом на документ или фотографию;
- `/setrole Telegram_ID participant|activist|leader|council|admin`;
- `/setstatus Telegram_ID new_member|involved_member|active_member|team_member|project_curator|community_leader`.
- `/setsetting имя_параметра https://новая-ссылка`.

## Структура

```text
app/
  handlers/       Telegram-сценарии участника, лидера, админа и чатов
  keyboards/      все клавиатуры
  states/         FSM регистрации, проектов, мероприятий, задач и вопросов
  services/       подписка, OpenAI, баллы, уведомления, расписание, аудит
  repositories/   запросы пользовательского домена
  database/       SQLAlchemy-модели и сессии
  utils/          тексты, константы, проверки ввода
alembic/          миграции
tests/            проверки схемы, домена, ввода и стиля текстов
```

## Локальная проверка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python -m unittest discover -v
python -m compileall -q app
```

Для реального локального запуска также нужны доступные PostgreSQL и Redis, после чего:

```bash
alembic upgrade head
python -m app.main
```

## Миграции

Первичная миграция создаёт полную схему. После изменения моделей создайте новую ревизию:

```bash
alembic revision --autogenerate -m "описание изменения"
alembic upgrade head
```

## Данные и безопасность

Бот хранит телефон, возраст, анкету и селфи-подтверждения. Для рабочего запуска необходимо:

- утвердить текст согласия и политику обработки данных с ответственным лицом организации;
- ограничить доступ к БД и резервным копиям;
- определить сроки хранения анкет, файлов и журналов;
- выдавать права администратора только доверенным аккаунтам;
- не публиковать `.env` и не передавать токены в чатах;
- настроить резервное копирование PostgreSQL;
- ограничить исходящий сетевой доступ контейнера там, где это возможно;
- проверить тексты и балльную систему на тестовой группе до публичного запуска.
